1.1 Java Overview & Architecture Module 1

What is Java?

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It was developed by James Gosling at Sun Microsystems (now owned by Oracle) and released in 1995.

The core philosophy of Java is WORA (Write Once, Run Anywhere). This means that compiled Java code can run on all platforms that support Java without the need for recompilation.

Quick Info: Why "Java"?
Java was originally named "Oak" after an oak tree that stood outside James Gosling's office. Later, it was renamed to "Green", and finally to "Java", named after the Peet's Coffee & Tea where the team often met. The magic number identifying a Java class file in hex is 0xCAFEBABE!

Key Features of Java

Java is popular because of these distinct features:

  1. Simple: Syntax is based on C++ but removes complex/dangerous features like explicit pointers and operator overloading.
  2. Object-Oriented: Everything in Java is an Object (except primitive data types). It follows OOP concepts like Inheritance, Abstraction, Polymorphism, and Encapsulation.
  3. Platform Independent (Architecture-Neutral): Java compiles to Bytecode, not machine code. This bytecode runs on the JVM, which is specific to the OS (Windows, Linux, Mac), making the code itself portable.
  4. Secure: Java runs inside a "sandbox" (JVM) and lacks pointers, preventing unauthorized memory access. It also has a Bytecode Verifier to check code fragments for illegal code.
  5. Robust: Strong memory management (Garbage Collection) and Exception Handling make Java reliable.
  6. Multithreaded: It allows concurrent execution of two or more parts of a program for maximum CPU utilization.

1.2 C++ vs Java: Syntax & Semantics Module 1

The Evolution

Java was derived from C++, so they share similar syntax (like loops, conditional statements, and class definitions). However, Java was designed to be simpler, safer, and platform-independent. The main design principle was to remove the features of C++ that were complex, unsafe, or rarely used.

Key Differences Table

Feature C++ Java
Platform Dependency Dependent: Compiled to machine code specific to the OS. Independent: Compiled to Bytecode, runs on any JVM.
Memory Management Manual: Developers use new/delete or malloc/free. Automatic: Garbage Collector (GC) handles memory freeing.
Pointers Supported: Allows direct memory manipulation (powerful but unsafe). Not Supported: Pointers exist internally but are not accessible to the programmer.
Multiple Inheritance Supported: A class can extend multiple classes. Not Supported (Classes): Solved using Interfaces.
Preprocessor Uses #include and #define. No preprocessor. Uses import and final variables.
Operator Overloading Supported. Not supported (except for + with Strings).

Detailed Semantic Differences

1. Memory Management: Manual vs Automatic

In C++, you are the janitor. In Java, you have a cleaning robot (Garbage Collector).

C++ (Manual) 
void dangerous() {
    int* ptr = new int(10);
    
    // ... logic ...
    
    // FORGOT TO DELETE! 
    // Result: Memory Leak
}
Java (Automatic) 
void safe() {
    Integer num = 10;
    
    // ... logic ...
    
    // No delete needed.
    // GC removes it when not used.
}

2. Pointers: Unsafe Arithmetic vs Safe References

Java prevents you from accidentally accessing memory you shouldn't touch.

C++ (Unsafe Pointers) 
int arr[3] = {1, 2, 3};
int* p = arr;

// SCARY: Accessing unknown memory!
// Could crash system or read secrets.
*(p + 100) = 99;
Java (References) 
int[] arr = {1, 2, 3};

// Java checks boundaries
// Throws Exception automatically
arr[100] = 99; // Crashing safely

3. The "Pass-by-Value" Trap

Common Confusion: "Since Java passes objects by reference, can I swap them easily?" No. Java passes the value of the reference (the address copy).

C++ (Pass by Ref) 
void swap(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}
// Calling swap(x, y) WORKS!
// x and y are actually changed.
Java (Pass Value of Ref) 
void swap(Integer a, Integer b) {
    Integer temp = a;
    a = b; // Only changes local 'a'
    b = temp;
}
// Calling swap(x, y) FAILS!
// Original x and y remain unchanged.
Quick Info: The "Goto" Statement
C++ supports the goto statement, which allows jumping to labeled parts of the code (often leading to "spaghetti code"). Java reserves goto as a keyword but does not implement it. You cannot use it in your code. Java uses break and continue with labels instead.

1.3 JVM, Compilation & Execution Module 1

The "Write Once, Run Anywhere" Process

Unlike C++, which compiles directly to machine code for a specific operating system, Java compiles into an intermediate format called Bytecode. This bytecode is platform-agnostic and can be executed by the JVM on any device.

Execution Steps

  1. Source Code (.java): The developer writes code in a text editor or IDE (e.g., Welcome.java).
  2. Compilation (javac): The Java Compiler (javac) checks syntax and translates the source code into Bytecode (Welcome.class).
    Command: javac Welcome.java
  3. Class Loading: The JVM doesn't load everything at once. It uses a ClassLoader Subsystem:
    • Bootstrap ClassLoader: Loads core Java libraries (java.lang.*, rt.jar).
    • Platform/Extension ClassLoader: Loads extensions (lib/ext).
    • Application ClassLoader: Loads your code (CLASSPATH).
  4. Bytecode Verification: The JVM acts as a gatekeeper. It checks the bytecode for security risks (e.g., unauthorized memory access) before execution.
  5. Execution (JVM): The Interpreter reads bytecode line-by-line, or the JIT (Just-In-Time) Compiler translates blocks of bytecode into native machine code for high performance.
    Command: java Welcome (Note: no .class extension here)

4. JVM Memory Structure: Stack vs Heap

Where does your data live? Understanding this is key to debugging.

Stack Memory
  • LIFO (Last-In-First-Out)
  • Stores methods and local primitive variables.
  • Small, extremely fast.
  • Cleared when method ends.
  • Error: StackOverflowError
Heap Memory
  • The "Pile" of Objects
  • Stores all Objects (new Dog()).
  • Larger, managed by Garbage Collector.
  • Objects live until cleaned.
  • Error: OutOfMemoryError

Writing Your First Program

Every Java application must contain a main method. Here is the standard structure: 

// File: HelloJava.java

public class HelloJava {
    // The entry point of the program
    public static void main(String[] args) {
        System.out.println("Welcome to Java Programming!"); 
    }
}

Dissecting public static void main

  • public: Access modifier. The JVM must be able to call this method from outside the class.
  • static: Allows the method to be called without creating an object of the class (the JVM starts before any objects exist).
  • void: The method does not return any value.
  • main: The specific name the JVM looks for to start execution.
  • String[] args: Stores command-line arguments passed to the program.
Quick Info: The Filename Rule
In Java, if a class is declared as public, the filename must match the class name exactly (case-sensitive).
If your class is public class MyCode, the file must be named MyCode.java. If you name it mycode.java, the compiler will throw an error!
Deep Dive: WORA (Write Once, Run Anywhere)
Is Java purely interpreted? No.
Early Java was purely interpreted and slow. Modern Java uses a Hybrid Model.
1. Compilation: Source -> Bytecode.
2. Interpretation: The JVM interprets bytecode initially.
3. JIT Compilation: The JVM monitors which parts of the code are run frequently (called "Hot Spots"). It compiles these hot spots into native machine code and caches them. This is why Java server applications often get faster the longer they run ("warming up").

Practical Pitfalls: When Data Types Bite Back

Choosing the wrong data type or ignoring limitations can lead to critical bugs.

1. Integer Overflow (The "Pac-Man" Effect)

What happens if you add 1 to the maximum possible integer? It wraps around to the minimum negative number! 

int max = 2147483647; // Maximum int value
System.out.println(max + 1); 
// Output: -2147483648 (Silent Overflow!)

Fix: Use long if you expect numbers to exceed 2 billion.

2. The Floating Point Error

Never use double or float for precise financial calculations. They use binary approximation. 

double a = 0.1;
double b = 0.2;
System.out.println(a + b);
// Output: 0.30000000000000004 (Not 0.3!)

Fix: Use BigDecimal for money.

3. Integer Division

System.out.println(5 / 2);   // Output: 2 (Truncated!)
System.out.println(5.0 / 2); // Output: 2.5

1.5 Basic Input & Output Module 1

1. Writing Output to Console

Java uses the System class to handle standard input and output streams. The most common way to print text is using System.out.

  • System.out.println("Text"); ‚Äî Prints text and moves the cursor to a new line.
  • System.out.print("Text"); ‚Äî Prints text but keeps the cursor on the same line.
  • System.out.printf("Age: %d", 20); ‚Äî Formatted output (similar to C's printf).

2. Reading Input (The Scanner Class)

To read input from the keyboard (System.in), we use the java.util.Scanner class. You must import this package to use it.

Common Scanner Methods

Method Description
nextInt() Reads an integer value.
nextDouble() Reads a floating-point number.
next() Reads a single word (stops at whitespace).
nextLine() Reads an entire line of text (including spaces).

Example: Interactive Program

import java.util.Scanner; // 1. Import the class

public class UserInput {
    public static void main(String[] args) {
        // 2. Create a Scanner object connected to keyboard input
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter your name: ");
        String name = sc.nextLine(); // Reads string

        System.out.print("Enter your age: ");
        int age = sc.nextInt(); // Reads integer

        System.out.println("Hello " + name + ", you are " + age + " years old.");

        sc.close(); // 3. Good practice: Close the scanner
    }
}
Quick Info: System.in vs System.out
Java provides three standard streams by default:
  • System.in: Standard Input Stream (usually the keyboard).
  • System.out: Standard Output Stream (the console screen).
  • System.err: Standard Error Stream. It is used to output error messages. In many IDEs (like Eclipse or IntelliJ), System.err prints in red text!
Common Pitfall: Scanner nextInt() + nextLine() Bug

This is a classic Java issue often called the "Scanner Skip" or "Leftover Newline" problem.

The short answer: nextInt() only reads the number, but it leaves the "Enter" key press (newline character) inside the input buffer.

The Breakdown

When you run code and type 25 and press Enter, two things go into the computer's input buffer:

  1. The number 25
  2. The newline character \n (created when you hit Enter)

The Sequence of Events:

  1. int age = sc.nextInt();
    • The scanner looks for an integer. It finds 25 and pulls it out.
    • Crucial Step: It stops reading right before the newline character.
    • Buffer status: \n (The newline is still sitting there waiting).
  2. String name = sc.nextLine();
    • The scanner looks for the next line of text.
    • It immediately sees the leftover \n from the previous step.
    • It thinks, "Oh, the user must have pressed Enter to finish this line!"
    • It consumes that newline and returns an empty string.
    • Result: The program skips letting you type a name.

The Fix

To fix this, you need to manually "consume" that leftover newline before asking for the name. You do this by adding an extra sc.nextLine() that you don't save to a variable.

System.out.print("Enter your age: ");
int age = sc.nextInt(); 

// FIX: Consume the leftover newline
sc.nextLine(); 

System.out.print("Enter your name: ");
String name = sc.nextLine();

Summary Table

Method What it reads What it does with the \n (Enter key)
nextInt() / nextDouble() Only the number Leaves it in the buffer (causing issues for the next read)
next() Only the next word Leaves it in the buffer
nextLine() The whole line Consumes it (removes it from buffer)
Code Complexity: Handling Invalid Input (Robustness)

What if you ask for an Age (int) but the user types "Twenty"? The program crashes with an InputMismatchException.

Real-world applications must check input before reading it. 

Scanner sc = new Scanner(System.in);
System.out.print("Enter your age: ");

// Check if the next input is actually an integer
if (sc.hasNextInt()) {
    int age = sc.nextInt();
    System.out.println("Age: " + age);
} else {
    System.out.println("Error: Please enter a valid number!");
    String badInput = sc.next(); // Consume the bad input so we don't get stuck
}
sc.close();
Best Practice: Automatic Resource Management

Since Java 7, you don't need to manually close scanners if you use try-with-resources.

// The "Old" Way (Risky)
Scanner sc = new Scanner(System.in);
// ... code ...
sc.close(); // What if an error crashes the code before this line? Memory Leak!

// The "Professional" Way (Safe)
try (Scanner sc = new Scanner(System.in)) {
    // ... code ...
} // Automatically closed here, even if it crashes!
Deep Dive: Performance (Scanner vs BufferedReader)

Scanner is convenient because it parses data (converts text to int/double) for you, but it uses Regular Expressions, which is slow for huge inputs.

BufferedReader simply reads text chunks. It is much faster but "dumber"‚Äîyou must parse string to numbers yourself.

Scanner (Easy) 
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
BufferedReader (Fast) 
BufferedReader br = new BufferedReader(
    new InputStreamReader(System.in)
);
int n = Integer.parseInt(br.readLine());

Verdict: Use Scanner for learning/small apps. Use BufferedReader for Competitive Programming (e.g., handling 1 million inputs).

1. The Paradigm Shift

Procedural programming (like C) focuses on functions and logic. Object-Oriented Programming (OOP) focuses on data and objects. It aims to model the software after real-world entities.

2. Class vs. Object

These are the fundamental atoms of OOP.

A. Encapsulation (The Shield)

Wrapping data (variables) and code (methods) together into a single unit. It involves hiding internal data from the outside world using private access modifiers and providing public Getters and Setters to access them.

B. Inheritance (The Family Tree)

The mechanism where one class acquires the properties and behaviors of another class. It promotes Code Reusability.
Keyword: extends

C. Polymorphism (Many Forms)

The ability of a method to do different things based on the object it is acting upon.

  • Runtime Polymorphism: Method Overriding.

D. Abstraction (The Surface)

Hiding complex implementation details and showing only the essential features of the object.
Achieved via: Abstract Classes and Interfaces.

Example: When you drive a car, you know how to use the steering wheel (Interface), but you don't need to know how the combustion engine works internally (Implementation).

Deep Dive: Composition vs. Inheritance
While Inheritance ("Is-A" relationship) is powerful, it creates tight coupling between classes.

Rule of Thumb: Favor Composition over Inheritance to make your code more flexible.

3.3 Constructors in Java Module 3

1. What is a Constructor?

A constructor is a special block of code used to initialize a newly created object. It looks like a method but has two specific rules:

  1. It must have the exact same name as the class.
  2. It must not have a return type (not even void).

It is called automatically when you use the new keyword.

2. Types of Constructors

A. Default Constructor (No-Argument)

If you do not define any constructor in your class, the Java compiler inserts a default one for you. It initializes member variables to their default values (0, null, false). 

class Bird {
    // Hidden default constructor provided by Java:
    // public Bird() { }
}

Bird b = new Bird(); // Works!

B. Parameterized Constructor

Used to initialize an object with specific values passed by the user. 

class Student {
    int id;
    String name;

    // Parameterized Constructor
    Student(int i, String n) {
        id = i;
        name = n;
    }
}

// Usage
Student s1 = new Student(101, "John");

3. Constructor Overloading

Just like methods, you can have multiple constructors in a single class, provided they have different parameter lists. This allows you to create objects in different ways. 

class Box {
    double width, height, depth;

    // Constructor 1: Cube
    Box(double len) {
        width = height = depth = len;
    }

    // Constructor 2: Custom Box
    Box(double w, double h, double d) {
        width = w;
        height = h;
        depth = d;
    }
}

Box cube = new Box(10);             // Uses Constructor 1
Box rectangle = new Box(10, 20, 30); // Uses Constructor 2

4. Copy Constructor

Unlike C++, Java does not have a built-in Copy Constructor, but you can create one to initialize an object using another object of the same class. 

Box(Box otherBox) {
    this.width = otherBox.width;
    this.height = otherBox.height;
    this.depth = otherBox.depth;
}
Quick Info: The "Void" Trap
If you accidentally add a return type to a constructor, it ceases to be a constructor and becomes a regular method! 
public class Test {
    // This is NOT a constructor. It is a method named "Test".
    public void Test() { 
        System.out.println("I am a method!");
    }
}
This code will compile, but the method will not run when you create a new Test().
Deep Dive: Constructor Chaining
You can call one constructor from another within the same class using this(). This is called Constructor Chaining and is used to reduce code duplication.

Rule: this() must be the first statement in the constructor body. 
class Employee {
    String name;
    int salary;

    Employee() {
        this("Unknown", 0); // Calls the parameterized constructor
    }

    Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }
}

3.4 Static, Final & Garbage Collection Module 3

1. The static Keyword

The static keyword means the member belongs to the class itself, rather than to any specific object instance.

Static Variables (Class Variables)

Shared among all objects. If one object changes the static variable, it changes for everyone.
Use case: Counters, Constants (like PI).

Static Methods

Can be called without creating an object. They can only access other static data; they cannot access non-static (instance) variables.
Use case: Utility functions like Math.sqrt(). 

class Counter {
    static int count = 0; // Shared memory

    Counter() {
        count++; // Increments the SHARED counter
    }
}

// Usage
new Counter();
new Counter();
System.out.println(Counter.count); // Output: 2

2. The final Keyword

The final keyword is used to restrict the user. It can be applied in three contexts:

Context Effect
Final Variable Value cannot be changed (Constant). final int MAX = 100;
Final Method Cannot be Overridden by subclasses.
Final Class Cannot be Inherited (Extended).

3. The Object Class

Every class in Java has a superclass. If you don‚Äôt extend any class, your class automatically extends java.lang.Object. It is the root of the class hierarchy.

Common methods inherited from Object:

  • toString(): Returns a string representation of the object.
  • equals(Object obj): Compares objects.
  • hashCode(): Returns a hash code value.
  • getClass(): Returns the runtime class of the object.

4. Garbage Collection (GC)

In C++, you must manually release memory. In Java, the Garbage Collector (GC) does this automatically.

How it Works

The GC runs in the background on the Heap memory. It identifies objects that are unreachable (no active reference points to them) and destroys them to free up memory. 

Student s1 = new Student();
s1 = null; // The Student object is now "unreachable" and eligible for GC.
Quick Info: Can we force GC?
You can request Garbage Collection using System.gc(), but you cannot force it. The JVM decides when it is actually necessary to run it based on available memory. It is merely a suggestion.
Deep Dive: The finalize() Method
Before an object is destroyed, the GC calls the finalize() method (similar to a Destructor in C++). You could override this to perform cleanup.
Note: Since Java 9, finalize() has been deprecated because it is unpredictable. Use try-with-resources or Cleaner classes instead.

4.1 Inheritance & Polymorphism Module 4

1. Inheritance Basics

Inheritance allows a new class (Subclass/Child) to adopt the attributes and methods of an existing class (Superclass/Parent). It represents an "Is-A" relationship (e.g., A Dog is an Animal).

Types of Inheritance in Java

  • Single Inheritance: Class B extends Class A.
  • Multilevel Inheritance: Class C extends Class B, which extends Class A.
  • Hierarchical Inheritance: Class B and Class C both extend Class A.
  • Multiple Inheritance: Not supported with classes (to avoid ambiguity). Supported via Interfaces.

2. Method Overriding

If a subclass provides a specific implementation of a method that is already provided by its parent class, it is called Method Overriding.

Rules:

  • Method name and parameters must be exactly the same.
  • Return type must be the same (or a subtype).
  • Access level cannot be more restrictive (e.g., cannot make a public method private). 
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

3. Dynamic Method Dispatch (Runtime Polymorphism)

This is the mechanism by which a call to an overridden method is resolved at runtime rather than compile-time. A reference variable of a Superclass can hold an object of a Subclass. 

Animal a = new Dog(); // Upcasting
a.sound(); // Output: "Dog barks"

Even though the reference type is Animal, the JVM detects that the actual object in memory is a Dog and runs the Dog's version of the method.

4. Abstract Classes

An abstract class is a class that cannot be instantiated. It is used to define a template.

  • Can have Abstract Methods (no body, must be implemented by subclass).
  • Can have Concrete Methods (regular methods with a body). 
abstract class Shape {
    abstract void draw(); // No body
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing Circle");
    }
}

5. The 'super' Keyword

The super keyword refers to the immediate parent class object.

  • super.variable: Access parent's variable.
  • super.method(): Call parent's method.
  • super(): Call parent's constructor.
Quick Info: The @Override Annotation
Using @Override above a method is technically optional, but highly recommended.
Why? If you make a typo (e.g., sounnd() instead of sound()), the compiler will throw an error telling you that you aren't actually overriding anything. Without the annotation, the compiler would just treat it as a new, separate method.
Deep Dive: Overloading vs. Overriding
Feature Overloading Overriding
Polymorphism Compile-time (Static) Runtime (Dynamic)
Signature Must be different Must be same
Classes Same class Parent & Child classes

4.2 Interfaces in Java Module 4

1. What is an Interface?

An interface is a reference type in Java. It is similar to a class but, unlike a class, it is a pure design contract. Until Java 8, an interface could only contain abstract methods and constants.

  • Keyword: Defined using interface.
  • Usage: A class implements an interface.
  • Relationship: Represents a "Can-Do" relationship (e.g., A Bird can fly).

2. Defining and Implementing

By default, all methods in an interface are implicitly public abstract, and all variables are public static final (constants). 

// Defining the Interface
interface Animal {
    void eat();   // Implicitly public abstract
    void sleep();
}

// Implementing the Interface
class Cat implements Animal {
    public void eat() {
        System.out.println("Cat eats fish.");
    }

    public void sleep() {
        System.out.println("Cat sleeps on the sofa.");
    }
}

3. Multiple Inheritance (The Solution)

Java does not allow a class to extend two classes (extends A, B is illegal). However, a class can implement multiple interfaces. This is how Java achieves Multiple Inheritance safely. 

interface Printable {
    void print();
}

interface Showable {
    void show();
}

// Implementing Multiple Interfaces
class Document implements Printable, Showable {
    public void print() { System.out.println("Printing..."); }
    public void show() { System.out.println("Showing..."); }
}

4. Extending Interfaces

An interface can extend another interface using the extends keyword (just like classes). 

interface A {
    void methodA();
}

interface B extends A { // B inherits methodA()
    void methodB();
}

class MyClass implements B {
    // Must implement BOTH methodA and methodB
}

5. Interface vs. Abstract Class

Feature Abstract Class Interface
Methods Can have abstract and concrete methods. Only abstract methods (pre-Java 8).
Variables Can have final, non-final, static, non-static. Only static final constants.
Inheritance A class can extend only ONE abstract class. A class can implement MULTIPLE interfaces.
Constructor Can have constructors. Cannot have constructors.
Quick Info: Marker Interfaces
A Marker Interface is an interface with no methods and no constants. It is empty!
Examples: Serializable, Cloneable.
Why? They tell the JVM that the object implementing them needs to be treated specially (e.g., "This object is allowed to be saved to a file").
Deep Dive: Java 8 Default Methods
Problem: If you add a new method to an Interface, all classes implementing it break because they haven't implemented the new method.
Solution (Java 8+): You can define default methods with a body inside an interface. This allows you to update interfaces without breaking existing code. 
interface Phone {
    void call();
    default void text() {
        System.out.println("Texting...");
    }
}

4.3 Packages in Java Module 4

1. What is a Package?

A package is a grouping mechanism (similar to a folder on your computer) used to organize related classes and interfaces. It prevents naming conflicts and controls access.

  • Built-in Packages: Provided by Java API (e.g., java.util).
  • User-defined Packages: Created by you.

2. Creating a Package

To put a class into a package, the very first line of your code must be the package statement. 

// File: Calculator.java
package mytools; // Creates a package named 'mytools'

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

Compiling: javac -d . Calculator.java
The -d . flag tells the compiler to create a folder named mytools and place the class file inside it automatically.

3. Using a Package (Importing)

To use a class from another package, you must import it. 

import mytools.Calculator; // Import specific class
// OR
import mytools.*; // Import EVERYTHING in the package

class Test {
    public static void main(String args[]) {
        Calculator c = new Calculator();
        System.out.println(c.add(10, 20));
    }
}

4. Standard Java Packages

The Java API is a library of prewritten classes. Here are the core packages you must know:

Package Description Common Classes
java.lang Core classes. Automatically imported (you don't need to import it). String, System, Math, Object, Thread
java.util Utility classes. Data structures and date/time tools. Scanner, ArrayList, Date, Random
java.io Input/Output. Dealing with files and streams. File, BufferedReader, InputStream
java.net Networking. Communicating over the internet. URL, Socket
Quick Info: Handling Name Conflicts
What if two packages have a class with the same name?
Example: java.util.Date and java.sql.Date.
You cannot import both. You must use the Fully Qualified Name for at least one of them: 
java.util.Date d1 = new java.util.Date();
java.sql.Date d2 = new java.sql.Date(System.currentTimeMillis());
Deep Dive: Static Imports
Since Java 5, you can import static members directly so you don't have to type the class name every time. 
import static java.lang.Math.*;

public class Test {
    public static void main(String[] args) {
        // No need to type Math.sqrt()
        System.out.println(sqrt(16)); 
        System.out.println(PI);
    }
}

4.4 Wrapper Classes, Enums & Metadata Module 4

1. Wrapper Classes

Java has a dual nature: it uses Primitive Types (for performance) and Objects (for OOP features).
Sometimes, you need to treat a primitive like an object (e.g., when saving an int into a ArrayList, which only accepts objects). Wrapper Classes "wrap" the primitive data type in an object.

Primitive Type Wrapper Class Description
byte Byte Object holding a byte
int Integer Object holding an int
char Character Object holding a char
double Double Object holding a double
boolean Boolean Object holding a boolean

2. Autoboxing & Unboxing

Before Java 5, converting between primitives and wrappers was manual and messy. Now, the compiler does it automatically.

  • Autoboxing: Automatic conversion of primitive to wrapper.
  • Unboxing: Automatic conversion of wrapper to primitive. 
// 1. Autoboxing (int -> Integer)
Integer numObj = 100; // Automatically converts int 100 to Integer object

// 2. Unboxing (Integer -> int)
int num = numObj; // Automatically extracts the value

3. Enumerations (Enums)

An enum is a special class that represents a group of constants (unchangeable variables). It is used when you have a predefined list of values (like Days, Colors, Status). 

// Defining an Enum
enum Level {
    LOW,
    MEDIUM,
    HIGH
}

public class TestEnum {
    public static void main(String[] args) {
        Level myVar = Level.MEDIUM;

        // Enums work perfectly with Switch statements
        switch(myVar) {
            case LOW:
                System.out.println("Low level");
                break;
            case MEDIUM:
                System.out.println("Medium level");
                break;
            case HIGH:
                System.out.println("High level");
                break;
        }
    }
}

4. Metadata (Annotations)

Metadata is data about data. In Java, this is handled via Annotations. They provide information to the compiler or runtime environment but do not directly affect code execution logic.

They always start with the @ symbol.

Common Built-in Annotations

  • @Override: Tells compiler this method is overriding a parent method.
  • @Deprecated: Marks a method/class as obsolete and warns users not to use it.
  • @SuppressWarnings("unchecked"): Tells compiler to ignore specific warnings.
Quick Info: Enums are Classes!
In C++, enums are just fancy integers. In Java, Enums are real Classes.
This means you can add constructors, methods, and variables inside an Enum! 
enum Size {
    SMALL(10), MEDIUM(20); // Enum constants calling constructor

    int volume;
    Size(int v) { this.volume = v; } // Constructor
}
Deep Dive: Integer Caching
Java caches Integer objects between -128 and 127. 
Integer a = 100;
Integer b = 100;
System.out.println(a == b); // TRUE (Same cached object)

Integer x = 200;
Integer y = 200;
System.out.println(x == y); // FALSE (New objects created)
This is a very common interview "gotcha" regarding wrapper classes!

5.1 Exception Handling Module 5

1. What is an Exception?

An exception is an abnormal condition that arises during the execution of a program (at runtime) which disrupts the normal flow of instructions.

2. The Exception Hierarchy

All exceptions descend from the java.lang.Throwable class.

  • Error: Serious problems that a reasonable application should not try to catch (e.g., OutOfMemoryError, StackOverflowError). Usually fatal.
  • Exception: Conditions that a reasonable application might try to catch.
    • Checked Exceptions: Checked at Compile-Time. You must handle them (e.g., IOException, SQLException).
    • Unchecked Exceptions (Runtime): Occur at Run-Time due to programming logic errors. You aren't forced to catch them (e.g., NullPointerException, ArithmeticException).

3. Keywords for Handling Exceptions

Keyword Usage
try Block of code to monitor for exceptions.
catch Block of code that executes if an exception occurs.
finally Block that always executes (used for cleanup like closing files), regardless of success or failure.
throw Used to explicitly throw an exception.
throws Declared in method signature to indicate the method might throw an exception.

4. Basic Try-Catch-Finally Block

public class TestException {
    public static void main(String[] args) {
        try {
            int data = 100 / 0; // Throws ArithmeticException
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero!");
        } finally {
            System.out.println("This executes no matter what.");
        }
    }
}

5. Nested Try Statements

You can have a try block inside another try block. If the inner catch doesn't handle the exception, the outer catch checks it. 

try {
    // Outer Block
    try {
        // Inner Block
        int b = 39 / 0;
    } catch (ArithmeticException e) {
        System.out.println("Inner: Div by zero");
    }
    
    int a[] = new int[5];
    a[10] = 50; // Array Index Out Of Bounds
    
} catch (Exception e) {
    System.out.println("Outer Catch: Generic error");
}

6. Creating Custom Exceptions

You can create your own exception by extending the Exception class. 

// 1. Define the Exception
class InvalidAgeException extends Exception {
    public InvalidAgeException(String s) {
        super(s);
    }
}

// 2. Use it
public class CustomTest {
    static void compute(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Not valid to vote");
        } else {
            System.out.println("Welcome to vote");
        }
    }
}
Quick Info: When does 'finally' NOT run?
The finally block executes even if you have a return statement in the try block!
The only time finally does not run is if you call System.exit(0) (which kills the JVM) or if the computer loses power/crashes.
Deep Dive: throw vs throws
This is a classic exam question.
  • throw: Used inside a method to actually throw an exception object.
    throw new IOException();
  • throws: Used in the method signature to declare that this method is dangerous.
    void myMethod() throws IOException { ... }

5.2 Multithreading in Java Module 5

1. What is Multithreading?

Multithreading is the process of executing multiple threads simultaneously. A Thread is a lightweight sub-process, the smallest unit of processing.
It allows a program to perform multiple tasks at the same time (e.g., typing in Word while the spell-checker runs in the background).

2. The Thread Life Cycle

A thread goes through five states during its life:

  1. New: Created but not yet started (new Thread()).
  2. Runnable: Ready to run and waiting for CPU time (start()).
  3. Running: The CPU is actively executing the thread.
  4. Non-Runnable (Blocked/Waiting): Paused (sleeping, waiting for I/O).
  5. Terminated: Execution finished.

3. Creating Threads

There are two ways to create a thread in Java.

Method A: Extending the Thread Class

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Test {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start(); // Internally calls run()
    }
}

Method B: Implementing the Runnable Interface (Recommended)

Better because you can still extend another class while implementing Runnable. 

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable thread is running...");
    }
}

public class Test {
    public static void main(String[] args) {
        MyRunnable r = new MyRunnable();
        Thread t1 = new Thread(r); // Pass the runnable to the Thread
        t1.start();
    }
}

4. Thread Synchronization

When multiple threads access shared resources (like a file or a variable) simultaneously, data corruption can occur. Synchronization ensures that only one thread accesses the resource at a time.

Keyword: synchronized

class Counter {
    int count = 0;
    
    // Only one thread can execute this method at a time
    public synchronized void increment() {
        count++;
    }
}

5. Inter-Thread Communication

Threads can talk to each other using three methods from the Object class (used inside synchronized blocks):

  • wait(): Pauses the current thread and releases the lock until notified.
  • notify(): Wakes up a single thread waiting on this object.
  • notifyAll(): Wakes up all waiting threads.

6. Thread Priorities

Every thread has a priority (1 to 10). The thread scheduler prefers higher priority threads.

  • Thread.MIN_PRIORITY (1)
  • Thread.NORM_PRIORITY (5) - Default
  • Thread.MAX_PRIORITY (10)
Thread t1 = new Thread();
t1.setPriority(Thread.MAX_PRIORITY);
Quick Info: start() vs run()
Question: Can you call run() directly instead of start()?
Answer: Yes, BUT it will not behave as a thread. It will run just like a normal method call in the main stack. You must call start() to actually create a new call stack and run it in parallel.
Deep Dive: Deprecated Methods
Your syllabus mentions "suspending/resuming threads".
In early Java, we had Thread.suspend() and Thread.resume().
Warning: These methods are now Deprecated (removed from modern use) because they are deadlock-prone. If a thread is suspended while holding a lock, no other thread can ever access that resource. Modern Java uses boolean flags or wait()/notify() to pause execution safely.

5.3 Networking in Java Module 5

1. The java.net Package

Java provides the java.net package to handle networking. It supports two primary protocols:

  • TCP (Transmission Control Protocol): Reliable, connection-oriented. Used for file transfers, chat apps, and web browsing. (Think: Phone call).
  • UDP (User Datagram Protocol): Unreliable, connection-less. Used for gaming and streaming where speed matters more than accuracy. (Think: Sending a letter).

2. TCP Programming (Socket Programming)

In TCP, a secure connection is established between the Client and the Server before data is sent.

  • ServerSocket: Used by the Server to listen for incoming connection requests.
  • Socket: Used by the Client to initiate a connection.

Example: Simple Server Concept

ServerSocket server = new ServerSocket(6666); // Open Port 6666
Socket s = server.accept(); // Wait for client... (Blocking call)

// Connection established! Now read/write streams.
DataInputStream dis = new DataInputStream(s.getInputStream());
String msg = dis.readUTF();
System.out.println("Client says: " + msg);

server.close();

3. UDP Programming (Datagrams)

UDP sends independent packets of data called Datagrams without establishing a connection.

  • DatagramSocket: Used to send or receive packets.
  • DatagramPacket: The container holding the data and the destination address. 
// Sending a packet via UDP
DatagramSocket ds = new DatagramSocket();
String str = "Hello UDP";
InetAddress ip = InetAddress.getByName("127.0.0.1");

DatagramPacket dp = new DatagramPacket(str.getBytes(), str.length(), ip, 3000);
ds.send(dp); // Fire and forget!
ds.close();

4. The URL Class

The URL (Uniform Resource Locator) class represents a web address. You can use it to download data from the web. 

import java.net.URL;

public class TestURL {
    public static void main(String[] args) {
        try {
            URL url = new URL("https://www.google.com");
            
            System.out.println("Protocol: " + url.getProtocol()); // https
            System.out.println("Host: " + url.getHost());       // www.google.com
            System.out.println("Port: " + url.getPort());       // -1 (Default)
        } catch (Exception e) { System.out.println(e); }
    }
}

5. URLConnection

The openConnection() method of the URL class returns a URLConnection object. This allows you to read the actual content (HTML) of the website.

Quick Info: Localhost
What is 127.0.0.1?
This is a special IP address known as localhost (or the loopback address). It always refers to "this computer."
When you write a client-server program on your laptop, the client connects to 127.0.0.1 to find the server running on the same machine.
Deep Dive: Ports
An IP address identifies the computer. A Port identifies the specific application running on that computer.
Ports are 16-bit numbers (0 to 65535).
  • 0-1023: Reserved (Well-known) ports. (e.g., HTTP uses 80, HTTPS uses 443).
  • 1024-49151: Registered ports (User apps).
  • 49152-65535: Dynamic/Private ports.

5.4 JDBC: Database Connectivity Module 5

1. What is JDBC?

JDBC (Java Database Connectivity) is a standard Java API that allows Java programs to connect to relational databases (like MySQL, Oracle, PostgreSQL) and execute SQL queries.
It acts as a bridge between your Java code and the Database.

[Image of JDBC Architecture diagram showing Java Application JDBC API JDBC Driver and Database]

2. The JDBC Architecture

To connect to a database, you need a specific Driver (a jar file provided by the database vendor, e.g., MySQL Connector).

  • DriverManager: Manages a list of database drivers.
  • Driver: Communicates with the specific database server.
  • Connection: Represents the active session with the database.
  • Statement: Used to send SQL queries.
  • ResultSet: Stores the data returned by the database.

3. The 5 Steps to Connect

Every JDBC program follows these five standard steps:

import java.sql.*;

public class DBConnect {
    public static void main(String[] args) {
        try {
            // 1. Load the Driver Class
            Class.forName("com.mysql.cj.jdbc.Driver"); 

            // 2. Create the Connection (URL, Username, Password)
            Connection con = DriverManager.getConnection(
                "jdbc:mysql://localhost:3306/mydb", "root", "password"
            );

            // 3. Create a Statement
            Statement stmt = con.createStatement();

            // 4. Execute Query & Process ResultSet
            ResultSet rs = stmt.executeQuery("SELECT * FROM students");

            while(rs.next()) {
                // Fetch data by column index or name
                System.out.println(rs.getInt(1) + " " + rs.getString(2));
            }

            // 5. Close the Connection
            con.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}

4. Statement vs. PreparedStatement

While Statement is simple, it is insecure for user input. You should almost always use PreparedStatement.

  • Statement: Compiles SQL every time. Vulnerable to SQL Injection.
  • PreparedStatement: Pre-compiled (Faster). Secure against SQL Injection. 
// Using PreparedStatement (Secure)
String query = "INSERT INTO students VALUES (?, ?)";
PreparedStatement pstmt = con.prepareStatement(query);

pstmt.setInt(1, 101);      // Set first ? to 101
pstmt.setString(2, "Ali"); // Set second ? to "Ali"

int i = pstmt.executeUpdate(); // Returns number of rows affected
Quick Info: execute() vs executeQuery()
  • executeQuery(): Used for SELECT statements. Returns a ResultSet.
  • executeUpdate(): Used for INSERT, UPDATE, DELETE. Returns an int (row count).
  • execute(): General purpose (rarely used unless you don't know the query type). Returns a boolean.
Deep Dive: SQL Injection
Why is Statement dangerous?
If you concatenate strings:
"SELECT * FROM users WHERE name = '" + userName + "'"
A hacker could enter the name: ' OR '1'='1
The query becomes: SELECT * FROM users WHERE name = '' OR '1'='1'
This is always true, so the database returns all user data (including passwords). PreparedStatement prevents this automatically.

2. The Applet Life Cycle

An Applet has a defined life cycle managed by the browser or the Applet Viewer. Understanding the order in which these methods are called is vital.

Method Purpose When is it Called?
init() First stage. Used for initialization (loading images, setting colors). Called only **once**. When the Applet is loaded.
start() Used to begin execution. Called **after init()** and every time the browser page is revisited. When the Applet becomes visible.
paint() Used to display output on the screen. Called when the applet needs redrawing. When the Applet is restored, minimized, or resized.
stop() Used to pause the execution. Called when the user leaves the page. When the Applet becomes invisible.
destroy() Final stage. Used to release resources and cleanup. Called only **once**. When the browser closes or the Applet is unloaded.

3. Layout Managers

Layout Managers determine how components are arranged inside a container. Without them, you would have to calculate exact (x, y) coordinates for every button!

A. FlowLayout (Default for Applets)

Arranges components left-to-right, top-to-bottom (like text in a paragraph). 

setLayout(new FlowLayout());
add(new Button("1"));
add(new Button("2"));

B. BorderLayout (Default for Frames)

Divides the window into 5 regions: North, South, East, West, and Center. 

setLayout(new BorderLayout());
add(new Button("North"), BorderLayout.NORTH);
add(new Button("Center"), BorderLayout.CENTER);

C. GridLayout

Arranges components in a grid of equal-sized cells (rows and columns). 

setLayout(new GridLayout(2, 2)); // 2 Rows, 2 Cols
add(new Button("1"));
add(new Button("2"));
add(new Button("3"));
add(new Button("4"));
Quick Info: Heavyweight vs. Lightweight
  • AWT Components (Heavyweight): When you create an AWT Button, Java asks Windows to create a real "Windows Button" or Mac to create a real "Mac Button". They look different on every OS.
  • Swing Components (Lightweight): Written entirely in Java. They look the same on all platforms and are drawn by Java itself.
Deep Dive: The repaint() method
You should never call paint() directly.
Instead, you call repaint().
Why? repaint() is a request to the JVM. The JVM schedules the update and calls update(), which clears the screen and then calls paint(). Calling paint() directly bypasses this lifecycle and causes graphical glitches.

6.3 Event Handling Module 6

1. The Delegation Event Model

In early Java, events were handled in a messy chain. Modern Java uses the Delegation Event Model. The concept is simple: A Source generates an event and sends it to one or more Listeners.

  • Source: The component where the action happens (e.g., a Button).
  • Event: An object describing what happened (e.g., "Button Clicked").
  • Listener: An interface containing the method to handle the event.

2. Common Event Listeners

To handle an event, a class must implement the appropriate Listener interface and override its methods.

Interface Event Class Methods to Override
ActionListener ActionEvent actionPerformed() (Buttons, Menus)
MouseListener MouseEvent mouseClicked(), entered(), exited(), pressed(), released()
KeyListener KeyEvent keyPressed(), keyReleased(), keyTyped()
WindowListener WindowEvent windowClosing(), windowOpened(), (and 5 others)

3. Example: Handling a Button Click

import java.awt.*;
import java.awt.event.*;

// 1. Implement the Listener Interface
public class ButtonDemo extends Frame implements ActionListener {
    TextField tf;

    ButtonDemo() {
        tf = new TextField();
        tf.setBounds(60, 50, 170, 20);
        
        Button b = new Button("Click Me");
        b.setBounds(100, 120, 80, 30);
        
        // 2. Register the Listener (tell the button "I am listening")
        b.addActionListener(this); 
        
        add(b); add(tf);
        setSize(300, 300);
        setLayout(null);
        setVisible(true);
    }

    // 3. Define what happens when clicked
    public void actionPerformed(ActionEvent e) {
        tf.setText("Welcome to Java Event Handling!");
    }

    public static void main(String args[]) {
        new ButtonDemo();
    }
}

4. Adapter Classes

Implementing an interface like WindowListener is annoying because you force-override all 7 methods, even if you only need windowClosing().
Adapter Classes solve this. They are abstract classes that implement the interface with empty methods. You extend the Adapter and override only what you need. 

// Instead of "implements WindowListener"
addWindowListener(new WindowAdapter() {
    public void windowClosing(WindowEvent e) {
        dispose(); // Close the frame
    }
});

5. Inner Classes & Anonymous Classes

Instead of making your main class implement interfaces, you can create Inner Classes or Anonymous Classes to handle events locally. This keeps code organized. 

Button b = new Button("Click");
// Anonymous Inner Class
b.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        System.out.println("Clicked!");
    }
});
Quick Info: getSource()
When you have multiple buttons handled by the same actionPerformed method, how do you know which one was clicked?
Use e.getSource(). 
if (e.getSource() == loginButton) { ... }
else if (e.getSource() == exitButton) { ... }
Deep Dive: Lambda Expressions (Java 8+)
Since Java 8, if an interface has only one method (Functional Interface) like ActionListener, you don't need anonymous classes. You can use Lambdas! 
b.addActionListener(e -> System.out.println("Clicked!"));
This reduces 5 lines of code to 1 line.

1. Java Swing (JFC)

Swing is part of the Java Foundation Classes (JFC). It is built on top of AWT but provides a richer set of GUI components.

2. Overview of Servlets

While Applets run on the Client (Browser), Servlets run on the Server.
A Servlet is a Java class used to extend the capabilities of a server that hosts applications accessed by a Request-Response model (like a Web Server).

[Image of Java Servlet Request Response architecture diagram]

Key Characteristics

  • Server-Side: Executes on the web server (e.g., Apache Tomcat).
  • Dynamic Content: Generates web pages (HTML) dynamically based on user input (e.g., search results, login pages).
  • Robust: Uses all features of Java (JDBC, Security, Multithreading).

The Servlet Life Cycle

  1. Loading: The Servlet class is loaded.
  2. init(): Initialization (called once).
  3. service(): Processes requests (GET/POST) and sends responses. Called for every user request.
  4. destroy(): Cleanup (called once before server shuts down).

Quick Info: The "J" Prefix
Why do all Swing components start with "J" (JButton, JFrame)?
It was done simply to distinguish them from the original AWT components (Button, Frame) so developers could mix them if absolutely necessary (though mixing is bad practice!).
Deep Dive: The Event Dispatch Thread (EDT)
Swing is not thread-safe. If you try to update the GUI (like changing a label text) from a background thread, the app might freeze or crash.
Rule: All GUI updates must happen on a special thread called the Event Dispatch Thread (EDT).
Use SwingUtilities.invokeLater(...) to safely update GUI from other threads.

56. Producer-Consumer (Synchronization) 
class Q {
    int n;
    boolean valueSet = false;

    synchronized int get() {
        while(!valueSet)
            try { wait(); } catch(InterruptedException e) {}
        System.out.println("Got: " + n);
        valueSet = false;
        notify();
        return n;
    }

    synchronized void put(int n) {
        while(valueSet)
            try { wait(); } catch(InterruptedException e) {}
        this.n = n;
        valueSet = true;
        System.out.println("Put: " + n);
        notify();
    }
}

class Producer implements Runnable {
    Q q;
    Producer(Q q) { this.q = q; new Thread(this).start(); }
    public void run() {
        int i = 0;
        while(true) q.put(i++);
    }
}

class Consumer implements Runnable {
    Q q;
    Consumer(Q q) { this.q = q; new Thread(this).start(); }
    public void run() {
        while(true) q.get();
    }
}
